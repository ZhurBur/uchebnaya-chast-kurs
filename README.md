### Запуск проекта

Проект будет корректно работать под управлением локального Node сервера. Для этого нужно установить на машину `Node.js` [https://nodejs.org/en/download](https://nodejs.org/en/download) и `npm` (`Node` сам предложит его установить во время инсталляции) в папке с проектом. После этого в той же командной строке выполнить команду установки `npm` пакета `http-server`:

`npm install http-server`

и дождаться установки. После этого нужно в этой же папке запустить сервер:

`http-server`

После этого в консоли отобразится локальный адрес приложения, скорее всего, это будет выглядеть так:

```
PS project_folder> http-server
Starting up http-server, serving ./
http-server version: 14.1.1

...

Available on:
  http://192.168.43.28:8080
  http://127.0.0.1:8080
  http://172.23.96.1:8080
  http://172.29.160.1:8080
Hit CTRL-C to stop the server
```

# Диаграммы системы "Учебная часть"

## 1. ДИАГРАММА ПРЕЦЕДЕНТОВ (USE CASE DIAGRAM)

### 1.1 Общее описание системы

**Система**: Система управления учебной частью образовательного учреждения

**Основные акторы (Действующие лица)**:
- **Администратор** - управляет системой (группы, студенты, преподаватели, предметы)
- **Преподаватель** - управляет студентами и оценками по своим предметам
- **Студент** - просматривает оценки и информацию о курсах

---

### 1.2 Нотация для диаграммы прецедентов

```
АКТОР: Администратор
├─ Прецедент: "Аутентификация"
│  └─ Описание: Вход в систему с логином и паролем
│  └─ Поток: Вводит учетные данные → Система проверяет → Перенаправляет на админ-панель
│
├─ Прецедент: "Управление группами"
│  ├─ Подпрецедент: "Просмотр всех групп"
│  │  └─ Описание: Видит список всех групп с информацией о предметах
│  │
│  ├─ Подпрецедент: "Создать новую группу"
│  │  └─ Поток: Кликает "Добавить группу" → Вводит название → Сохраняет
│  │  └─ Проверка: Название должно быть уникальным
│  │
│  ├─ Подпрецедент: "Редактировать группу"
│  │  └─ Поток: Кликает на группу → Выбирает предметы через чекбоксы → Сохраняет
│  │
│  └─ Подпрецедент: "Удалить группу"
│     └─ Проверка: Группа должна быть пуста (без студентов)
│     └─ Поток: Кликает "Удалить" → Подтверждает → Удаляется
│
├─ Прецедент: "Управление студентами"
│  ├─ Подпрецедент: "Просмотр всех студентов"
│  │  └─ Описание: Видит список с ФИО, группой, логином
│  │
│  ├─ Подпрецедент: "Добавить студента"
│  │  └─ Поток: Кликает "Добавить студента" → Вводит ФИО, выбирает группу, логин, пароль → Сохраняет
│  │  └─ Проверка: Логин должен быть уникален, группа должна существовать
│  │
│  ├─ Подпрецедент: "Изменить группу студента"
│  │  └─ Поток: Выбирает новую группу из выпадающего списка → Система сохраняет
│  │
│  └─ Подпрецедент: "Удалить студента"
│     └─ Побочный эффект: Удаляются все оценки студента
│     └─ Поток: Кликает "Удалить" → Подтверждает → Удаляется с оценками
│
├─ Прецедент: "Управление преподавателями"
│  ├─ Подпрецедент: "Просмотр преподавателей"
│  │  └─ Описание: Видит список преподавателей (статичные данные, не редактируются)
│  │  └─ Информация: ID, ФИО, кафедра, логин
│  │
│  └─ Подпрецедент: "Назначить предметы преподавателю"
│     └─ Поток: Кликает на ФИО преподавателя → Выбирает предметы через чекбоксы → Сохраняет
│
├─ Прецедент: "Управление предметами"
│  ├─ Подпрецедент: "Просмотр всех предметов"
│  │  └─ Описание: Видит список предметов с часами/кредитами
│  │
│  ├─ Подпрецедент: "Добавить предмет"
│  │  └─ Поток: Кликает "Добавить предмет" → Вводит название, часы, кредиты → Сохраняет
│  │  └─ Проверка: Часы и кредиты должны быть положительными числами
│  │
│  ├─ Подпрецедент: "Редактировать предмет"
│  │  └─ Поток: Кликает на предмет → Выбирает преподавателей через чекбоксы → Сохраняет
│  │
│  └─ Подпрецедент: "Удалить предмет"
│     └─ Проверка: По предмету не должно быть оценок
│     └─ Поток: Кликает "Удалить" → Подтверждает → Удаляется
│
└─ Прецедент: "Выйти из системы"
   └─ Поток: Кликает "Выйти" → Сессия очищается → Перенаправляется на страницу входа


АКТОР: Преподаватель
├─ Прецедент: "Аутентификация"
│  └─ Описание: Вход в систему с логином и паролем
│
├─ Прецедент: "Просмотр студентов"
│  └─ Описание: Видит список своих студентов по двум категориям
│  ├─ Подпрецедент: "Просмотр студентов со оценками"
│  │  └─ Описание: Студенты, которым уже поставлены оценки по предметам преподавателя
│  │
│  └─ Подпрецедент: "Просмотр студентов без оценок"
│     └─ Описание: Студенты, которым еще не поставлены оценки по предметам преподавателя
│
├─ Прецедент: "Просмотр профиля студента"
│  └─ Поток: Кликает на имя студента → Видит его профиль
│  └─ Информация: ФИО, группа, логин
│
├─ Прецедент: "Выставление оценок"
│  └─ Описание: Выставляет оценки студентам по своим предметам
│  ├─ Подпрецедент: "Поставить оценку"
│  │  └─ Поток: Открывает профиль студента → Выбирает оценку из выпадающего списка (2-5) → Автосохранение
│  │  └─ Возможные оценки: 2 (Неуд.), 3 (Удовл.), 4 (Хорошо), 5 (Отлично)
│  │
│  ├─ Подпрецедент: "Изменить оценку"
│  │  └─ Поток: Выбирает новую оценку → Автосохранение
│  │
│  └─ Подпрецедент: "Удалить оценку"
│     └─ Поток: Выбирает пустой вариант в выпадающем списке → Оценка удаляется
│
├─ Прецедент: "Удаление студента"
│  └─ Описание: Удаляет студента из системы (если необходимо)
│  └─ Побочный эффект: Удаляются все оценки студента
│
└─ Прецедент: "Выйти из системы"
   └─ Поток: Кликает "Выйти" → Сессия очищается


АКТОР: Студент
├─ Прецедент: "Аутентификация"
│  └─ Описание: Вход в систему с логином и паролем
│
├─ Прецедент: "Просмотр личной информации"
│  └─ Описание: Видит свои ФИО и номер группы
│
├─ Прецедент: "Просмотр оценок"
│  └─ Описание: Видит таблицу всех своих оценок по предметам
│  └─ Информация: Предмет, преподаватель, часы/кредиты, оценка
│  └─ Фильтрация: Показывает только предметы группы или предметы с оценками
│  └─ Визуализация: Оценки окрашены (5=зелёный, 4=синий, 3=оранжевый, 2=красный)
│
└─ Прецедент: "Выйти из системы"
   └─ Поток: Кликает "Выйти" → Сессия очищается


АКТОР: Система (внутренняя)
├─ Прецедент: "Инициализация БД"
│  └─ Описание: При первом запуске создает начальные данные в localStorage
│  └─ Создаёт: Админа, демо-студента, группы, предметы, счетчики ID
│
├─ Прецедент: "Миграция данных"
│  └─ Описание: При обновлении системы добавляет новые поля к существующим данным
│
└─ Прецедент: "Валидация данных"
   └─ Описание: Проверяет корректность введенных данных перед сохранением

```

   - Поддерживает все типы диаграмм UML

2. **Draw.io** (https://draw.io)
   - Визуальный редактор
   - Экспорт в разные форматы

3. **Lucidchart** (https://www.lucidchart.com/)
   - Профессиональные диаграммы
   - Совместное редактирование

4. **Miro** (https://miro.com/)
   - Для командной работы
   - Интеграция с другими инструментами

- Преобразования в PlantUML синтаксис

- Описания во время презентации
